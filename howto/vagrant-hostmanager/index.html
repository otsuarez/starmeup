A Vagrant plugin that manages the /etc/hosts file on guests within a multi-machine environment

install

install the plugin vagrant-hostmaster

vagrant plugin install vagrant-hostmanager

$ vagrant plugin install vagrant-hostmanager
Installing the 'vagrant-hostmanager' plugin. This can take a few minutes...
Installed the plugin 'vagrant-hostmanager (1.5.0)'!
$

issues

https://developerkarma.com/2014/03/19/vagrant-tricks.html

There was an error executing ["sudo", "/usr/local/bin/vagrant-lxc-wrapper", "lxc-create", "--version"]

For more information on the failure, enable detailed logging by setting
the environment variable VAGRANT_LOG to DEBUG.

solution

It is a known issue with the vagrant-lxc plugin and a solution is proposed on the CHANGELOG file.

Remove plugin version from config file name generated by the vagrant lxc sudoers command. Manual removal of /usr/local/bin/vagrant-lxc-wrapper-* / /etc/sudoers.d/vagrant-lxc-* files are required.

Remove the vagrant-lxc-wrapper command (just renaming it will work).

sudo mv /usr/local/bin/vagrant-lxc-wrapper /usr/local/bin/vagrant-lxc-wrapper.disabled

working

A vagrant up is required for the plugin to work.

When the vm boots up, you will notice the following messages on the boot messages

==> lb: Machine booted and ready!
==> lb: Setting hostname...
==> lb: Updating /etc/hosts file on active guest machines...
==> lb: Updating /etc/hosts file on host machine (password may be required)...

The /etc/hosts file on each vm will be updated accordingly.

## vagrant-hostmanager-start
10.1.2.202  vagrant-nms
10.1.2.26 lb
## vagrant-hostmanager-end
