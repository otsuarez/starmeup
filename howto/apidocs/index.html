what is it

apiDoc creates a documentation from API descriptions in your source code, which can be in Java, JavaScript, PHP, CoffeeScrip, Erlang, Perl, Python, Ruby, Go, Dart, ....

three steps to documenting

  • Install apidoc in your development workstation.
  • Add inline documentation to your code in Javadoc-style, using a simple syntax.
  • execute the apidoc command to create/update the documentation.
  • like said, no four step, the documentation is ready to be browsed.

Installation

Nodejs is a requirement.

sudo npm install apidoc -g

Usage

apidoc -i myapp/ -o apidoc/ -t mytemplate/

That command will create an apiDoc of all files within dir myapp/, uses template from dir mytemplate/ and put all output to dir apidoc/.

Without any parameter, apiDoc generate a documentation from all .cs .dart .erl .go .java .js .php .py .rb .ts files in current dir (incl. subdirs) and writes the output to ./doc/.

The input directory could be the current working directory, from where the command is being execute: -i ..

Configuration (apidoc.json)

The optional apidoc.json in your projects root dir includes common information about your project like title, short description, version and configuration options like header / footer settings or template specific options.
